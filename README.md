## 🚀 Setup in 5 Minutes

### Step 1 — Get Your Free Groq API Key

1. Open **https://console.groq.com**
2. Click **Sign Up** (use Google or GitHub — no credit card)
3. Click **"API Keys"** in the left sidebar
4. Click **"Create API Key"**
5. Copy the key (starts with `gsk_`)

---

### Step 2 — Get Free MongoDB URI

**Option A — Cloud (Recommended):**
1. Open **https://cloud.mongodb.com**
2. Click **"Try Free"** and sign up
3. Create a **Free M0 cluster** (any region, click through defaults)
4. Go to **"Database Access"** → Add a user with username + password
5. Go to **"Network Access"** → Add IP `0.0.0.0/0` (allow everywhere)
6. Go to **"Database"** → Click **"Connect"** → **"Drivers"**
7. Copy the connection string, replace `<password>` with your DB password

**Option B — Local MongoDB:**
```
MONGODB_URI=mongodb://localhost:27017/ai-tutor-os
```

---

### Step 3 — Configure Environment Files

**Backend:**
```bash
cd backend
cp .env.example .env
```
Open `backend/.env` and fill in:
```
GROQ_API_KEY=gsk_your_key_here
MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/ai-tutor-os
```

**Frontend:**
```bash
cd frontend
cp .env.local.example .env.local
```
The default `NEXT_PUBLIC_API_URL=http://localhost:5000` is correct for local dev.

---

### Step 4 — Install & Run

```bash
# In the root ai-tutor-os/ folder:

npm run install:all    # installs all packages (takes ~1-2 min)

npm run dev            # starts both frontend + backend
```

- **Frontend** → http://localhost:3000
- **Backend**  → http://localhost:5000
- **Health**   → http://localhost:5000/api/health

## 🧠 How the AI Works

```
User uploads PDF / pastes notes
           ↓
Backend extracts text (pdf-parse)
           ↓
Groq (llama-3.3-70b) generates lesson:
   • Title & summary
   • 5 key points
   • Main explanation
   • Real-world Indian example
   • Analogy
   • Key terms
   • Teaching script (for voice)
   • Quick recap
           ↓
Student reads lesson
           ↓
Clicks "I Don't Understand" (up to 5 times)
   → Attempt 1: Story-based (Indian characters)
   → Attempt 2: Step-by-step visual
   → Attempt 3: 3 real-world examples
   → Attempt 4: Simple analogy
   → Attempt 5: Q&A format
           ↓
Student chats → Groq (llama-3.1-8b, ultra-fast) replies
           ↓
Quiz generated by Groq → Student submits → Score + explanations shown
           ↓
Voice: Browser Web Speech API speaks the lesson (FREE, no API)
```

## 🔧 Troubleshooting

| Problem | Fix |
|---------|-----|
| `GROQ_API_KEY not set` | Add key to `backend/.env` |
| `Session not found` | Backend is not running — run `npm run dev:backend` |
| PDF text empty | PDF might be image-only (scanned). Use "Paste Notes" tab instead |
| CORS error | Make sure `FRONTEND_URL=http://localhost:3000` in `backend/.env` |
| Voice not working | Use Chrome or Edge browser (Firefox sometimes works too) |
| MongoDB error | App still works without DB — sessions are stored in memory |

## 📞 API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/upload/file` | Upload PDF/TXT file |
| POST | `/api/upload/text` | Upload raw text/notes |
| POST | `/api/lesson/generate` | Generate AI lesson |
| GET  | `/api/lesson/session/:id` | Get session + lesson |
| POST | `/api/lesson/reexplain` | Re-explain in new style |
| POST | `/api/chat/message` | Chat with AI tutor |
| GET  | `/api/chat/history/:id` | Get chat history |
| POST | `/api/quiz/generate` | Generate quiz |
| POST | `/api/quiz/submit` | Submit + score quiz |
| GET  | `/api/health` | Status check |
